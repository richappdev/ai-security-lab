README Description

How to run and use the lab

This project provides a local Docker environment for building and testing security tools safely.

Requirements

  • Docker Desktop
  • Docker Compose v2
  • PowerShell
  • Python for local unit tests

Included Targets

  • OWASP Juice Shop at http://127.0.0.1:3000
  • DVWA at http://127.0.0.1:8080
  • Security API at http://127.0.0.1:8000

Start Commands

powershell -ExecutionPolicy Bypass -File .\scripts\check-prereqs.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\start-lab.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\validate-lab.ps1

Example API Calls

Invoke-RestMethod `
  -Method Post `
  -Uri http://127.0.0.1:8000/scan/passive/headers `
  -ContentType application/json `
  -Body '{"target":"http://juice-shop.local:3000","operator":"local-user"}'

Invoke-RestMethod `
  -Method Post `
  -Uri http://127.0.0.1:8000/scan/active/http-methods `
  -ContentType application/json `
  -Body '{"target":"http://juice-shop.local:3000","operator":"local-user","rate_limit_per_minute":30}'

Operating Rule

Browser access uses localhost URLs. API scan requests use Docker network aliases such as juice-shop.local because the API container makes the target request from inside the lab network.
